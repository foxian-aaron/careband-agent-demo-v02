# 风险规则说明

## 风险等级

- `data_insufficient`：数据不足
- `stable`：稳定
- `observation`：观察
- `attention`：需关注
- `high_risk`：高风险
- `urgent`：紧急

## 分数映射

- 0-24：稳定
- 25-44：观察
- 45-69：需关注
- 70-89：高风险
- 90-100：紧急

硬事件优先于普通分数映射和数据不足判断。组合高风险至少为 `high_risk`，但不直接替代 SOS 等紧急事件。

## 规则 1：数据不足

如果没有 SOS、跌倒、越界或严重主诉等硬事件，且 `dataCompleteness < 0.4`，输出“数据不足”，风险分不超过 30，关键原因包含“今日数据完整度不足，需先确认设备佩戴或数据同步”。

## 规则 2：步数明显下降

如果 `stepsToday < avgSteps7d * 0.5`，活动状态为 `significantly_low`，加 20 分，原因说明今日步数低于本人 7 日平均约多少百分比。

## 规则 3：睡眠偏低

如果 `sleepDuration < avgSleep7d - 1.5`，睡眠状态为 `below_baseline`，加 15 分，原因说明低于本人基线多少小时。

## 规则 4：晚药未确认

如果 `medicationEvening = not_confirmed`，用药状态为 `not_confirmed`，加 15 分。

晚药未确认是风险加权因素之一，单独不等于高风险。它需要与活动明显下降、睡眠偏低、主诉不适等信息组合判断，才会提升到更高优先级。

## 规则 5：主诉症状

如果事件中存在 `voice_symptom`，且原始文本包含“头晕、不舒服、胸闷、心口痛、摔倒、走不动、喘不过气”等关键词，加 25 分，并保留原话。

## 规则 6：慢病放大因子

如果老人有“高血压”或“冠心病史”，且存在主诉不适事件，加 10 分，并记录“慢病标签 + 主诉不适，提升关注优先级”。

## 规则 7：组合高风险

如果同时满足活动明显下降、晚药未确认、主诉包含头晕/不舒服/胸闷，则风险等级至少为 `high_risk`，建议护工立即查看。

## 规则 8：SOS 或跌倒

如果存在 SOS 事件，风险等级直接为 `urgent`，风险分至少 90。即使 `dataCompleteness < 0.4`，也先按 SOS 应急流程处理，并在原因中补充“虽然今日数据完整度不足，但老人触发了 SOS，需优先按应急流程处理。”

如果检测到跌倒事件，风险等级至少为 `high_risk`。如果 `payload.noResponseSeconds >= 30`，升级为 `urgent`。

如果存在 `location_alert` 且 `payload.safeZoneStatus = "outside"`，风险等级至少为 `high_risk`，建议护工确认位置。

如果 `voice_symptom` 包含“救命、摔倒了、胸口痛、喘不过气、动不了”等严重关键词，直接进入紧急流程。

## 规则 9：稳定

如果没有明显偏离，风险等级为 `stable`，原因包含“今日状态接近本人近期基线”。

## 规则 10：观察和需关注

轻度活动下降、夜间离床增加、睡眠偏低等会进入观察或需关注区间。

## 前台展示状态

`riskLevel` 只表示今日风险判断。护工处理后不会把今日历史高风险改成稳定，而是通过 `deriveDisplayStatus` 输出前台展示状态：

- 高风险 + 待处理：高风险待处理
- 高风险 + 处理中：高风险处理中
- 高风险 + 已查看：已查看 / 待完成记录
- 高风险 + 完成：已跟进 / 持续观察

## 非医疗诊断边界

所有建议动作只能写“建议护工查看”“建议持续观察”“建议联系专业人员判断”“按机构应急流程处理”等表达。不得输出医疗结论、确诊类措辞或猜测性疾病判断。
