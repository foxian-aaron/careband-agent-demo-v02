# 智护环 CareBand Agent Demo v0.1.3 路演脚本

## 1. 开场介绍

大家好，我们展示的是“智护环 CareBand Agent”，一个面向长者照护场景的 AI Agent 状态观察系统。它不是普通健康 Dashboard，也不是聊天机器人，而是把长者状态、个人基线、照护事件和护工处理串成一个闭环。

## 2. 展示机构端

打开 `#/institution`。这里有 5 位长者的风险热力图，管理者可以看到“当前未闭环高风险”“今日曾高风险”“已跟进高风险”“待处理任务”和“平均数据完整度”。这样不会把“今日曾发生过高风险”和“当前仍未处理的高风险”混在一起。

## 3. 点击陈伯

点击陈伯进入 `#/elder/E001`。陈伯 78 岁，房间 203，有高血压标签，平时 7 日平均步数 2150 步，平均睡眠 6.5 小时。

## 4. 说明陈伯初始“需关注”

今天陈伯只有 820 步，睡眠 4.8 小时，晚药未确认。系统不是用通用阈值判断，而是和陈伯自己的个人基线比较，因此初始前台展示状态为“需关注”，今日风险等级为 attention。

## 4.5 展示 Decision Trace

在驾驶舱下方展示 Decision Trace。它分成输入数据、个人基线比较、触发规则、风险输出和三端输出五部分，说明 Agent 不是黑盒，而是从结构化数据和规则链路生成多角色摘要。

## 4.6 展示老人档案页

点击“查看老人档案”，进入 `#/elder/E001/profile`。

讲解词：

“这里可以看到陈伯的基础档案、慢病标签、用药风险、照护团队和授权状态。系统不是只看当天数据，而是结合长期档案和个人基线判断今天是否偏离平时。”

强调：慢病标签和风险标签只是照护参考，不是医疗诊断。

## 4.7 展示用药计划页

点击“查看用药计划”，进入 `#/medication/E001`。

讲解词：

“这里展示今日早药和晚药计划。陈伯早药已确认，晚药尚未确认。当晚药未确认叠加活动下降和头晕反馈时，系统会提高护工关注优先级。确认晚药后，家属端和机构端会同步更新。”

## 5. 触发“我有点头晕”

进入 `#/demo-control`，点击“触发陈伯头晕语音事件”。系统添加 20:15 的 voice_symptom 事件，原始文本是“我有点头晕”。

## 6. 展示状态升级为高风险

回到陈伯驾驶舱。Decision Trace 展示规则触发链路：活动明显下降、睡眠偏低、晚药未确认、老人主动反馈头晕，再叠加慢病标签，陈伯变为“高风险待处理”。

## 7. 展示护工待办

打开 `#/caregiver`。护工端出现高优先级任务“陈伯需要立即查看”，原因是“头晕反馈 + 晚药未确认 + 活动明显下降”，建议护工确认是否已进食和服药，并观察不适是否持续。

## 8. 展示家属安心卡

打开 `#/family/E001`。家属端不会展示复杂医学指标，而是温和说明：今晚 20:15 陈伯反馈有点头晕，系统已提醒护工，陈伯仍在长者中心内，未检测到跌倒。

## 9. 护工接单

在 Demo 控制台或护工端点击“护工接单”。任务状态变为处理中，家属端会看到“护工已接单，正在查看陈伯情况”。

## 10. 护工标记已查看

点击“标记已查看”。时间线新增 caregiver_checked 事件，表示接单之后护工已到场确认，不再把接单和查看混为一件事。

## 11. 确认晚药

可以在护工端点击“确认晚药”，也可以进入 `#/medication/E001` 点击确认。陈伯晚药状态变为已确认，家属端显示“晚药已确认”，机构端同步更新用药维度。

## 12. 完成处理

点击“完成护工处理”。系统写入备注：“20:25 护工A已查看陈伯，已确认晚药，陈伯目前在房间休息，建议明早继续关注活动和睡眠。”

## 13. 展示三端状态同步

回到家属端，主标题显示“陈伯今日状态：已跟进 / 持续观察”。回到机构端，陈伯前台展示状态为“已跟进 / 持续观察”，今日风险等级仍显示“高风险”，并说明今日曾发生高风险事件但已经处理。此时机构端“当前未闭环高风险”为 0，“今日曾高风险”为 1，“已跟进高风险”为 1。

## 14. 总结产品价值

智护环的核心价值是把长者贴身状态感知、个人基线对比、多指标风险判断、三端照护展示和护工处理闭环连接起来。系统不替代医生，只做照护风险提示和任务分派，帮助照护人员更早发现需要关注的人和事。
